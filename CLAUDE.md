## Project overview

Nightfall is a self-hosted sleep analytics platform that imports Samsung Health data (Android → FastAPI → PostgreSQL) and visualizes it as a personal health dashboard. The central architectural bet is **local-first with zero cloud dependency for health data**, which means every component that touches Art.9 RGPD data runs on the user's own VPS or local machine. The app discovered a Non-24 circadian disorder that 15 years of medical consultations missed — pattern visibility over raw numbers is the core product bet.

---

## Architecture

**Key components:**
- `server/` — FastAPI async backend: routers by domain, SQLAlchemy models, structlog, Argon2id auth, AES-256-GCM field encryption
- `server/security/` — crypto, rate limiting, CSRF, RGPD (export/erase/audit-log), JWT, Argon2id
- `server/routers/` — REST endpoints: `auth`, `sleep`, `heartrate`, `steps`, `exercise`, `mood`, `me`, `admin`, `health`
- `alembic/` — schema migrations (always `upgrade head` in deploy pipeline)
- `static/` — frontend HTML/JS/CSS served by FastAPI (WebView-compatible, no build step)
- `android-app/` — Kotlin Jetpack Compose (not started; Phase 4)
- `agents/contracts/` — Pydantic schemas used by skills to generate `brief.json` for agent inputs
- `docs/vault/specs/` — Obsidian specs (source of truth for features)
- `docs/vault/code/` — vault code notes generated by code-cartographer

**Data flow:**
1. Android SAF picks a Samsung Health ZIP → `POST /sleep` (or similar router) → FastAPI validates; no decryption on ingestion — encryption occurs on write (step 2).
2. FastAPI service encrypts Art.9 fields (AES-256-GCM via `server/security/crypto.py`) before writing to PostgreSQL.
3. Dashboard JS (`static/`) calls authenticated REST endpoints → renders visualizations in-browser (or WebView in Android).

## Conventions

**Backend (Python 3.12 + FastAPI):**
- Routers: one file per domain in `server/routers/`. No business logic in routers — delegate to service functions.
- Models: all Art.9 fields use `EncryptedField`. Adding a plain `String` for health data = violation of C2.
- Async everywhere: `async def` for all route handlers and DB operations.
- Logging: `structlog` only — never `print()`, never `logging.getLogger()` directly.
- Tests: `pytest` + real DB (no mocking the database — mock/prod divergence burned us before).

**Naming:**
- Files: `snake_case.py`
- Routes: `kebab-case` URL paths, `snake_case` Python handlers
- Alembic migrations: `<YYYY>_<seq>_<short_desc>.py` (autogenerate handles this)

**File placement:**
- New domain endpoints → `server/routers/<domain>.py`
- New security utilities → `server/security/<concern>.py`
- Shared DB utilities → `server/database.py` or `server/db/`
- Frontend assets → `static/` (no build step, vanilla JS/CSS)
- Android → `android-app/` (Phase 4+)

**Commit style:** single-line conventional commits. No multi-line bodies, no bullet points, no `Co-authored-by` trailers.

## Constraints — non-negotiable

### C1 — Local-first absolu
Health data (Art.9 RGPD) never leaves user-controlled infrastructure. Never add:
- External API calls that send health data (no Firebase, Supabase, AWS, GCP, PaaS)
- Cloud storage for raw health fields
- Third-party analytics SDKs that could leak health metadata

### C2 — RGPD santé niveau maximum
Every Art.9 field must be AES-256-GCM encrypted at rest. Any new model field storing health data needs `EncryptedField` — ask before using plain `String`/`Integer`. The `export`, `erase`, and `audit-log` endpoints are load-bearing for legal compliance — do not break their contracts.

### C3 — Sécurité intégrée
`pentester` agent runs on every spec and every PR via `/review`. A finding of severity ≥ HIGH blocks merge — it is not optional. Do not bypass with `--no-verify` or by skipping the review step.

### Design — DataSaillance tokens only
Token source of truth: `../Vectorizer/IdentiteVisuelle/` (logos + polices). OpenDesign is referenced for CSS structure only, not for color values. Dark mode palette: bg `#191e22`, surface `#232e32`, teal `#0e9eb0`, amber `#d37c04`, cyan `#3be5e7`. Light **and** dark mode are both mandatory. Forbidden: `#6366f1` (indigo Tailwind), `linear-gradient` decorative, box-shadow glow effects, font other than Cairo.

### No LLM on health data
Nightfall is explicitly not an LLM wrapper. Do not add Claude/OpenAI/etc. calls that receive raw health fields as input. Visualization and pattern detection are algorithmic only.

### Branch discipline
Never commit to `main` or `dev` directly. Always: `feat/<scope>`, `fix/<scope>`, `chore/<scope>`.

## Agent/skill system

Skills generate a `brief.json` using Pydantic schemas in `agents/contracts/`. Agents read this brief from `${CLAUDE_PROJECT_DIR}/${WORK_DIR}/brief.json`. Before modifying a skill invocation chain, check the relevant contract in `agents/contracts/`.

**Invocation chain for a new feature:**
```
/spec <slug>          → spec-writer writes docs/vault/specs/<date>-spec-<slug>.md
/vision <spec-path>   → vision-keeper audits spec vs VISION.md (aligned/drift_alert)
/tdd <spec-path>      → test-writer writes RED tests in tests/
/impl <spec-path>     → coder-backend makes tests GREEN
/review               → reviewer + pentester audit diff
/commit               → git-steward handles git ops + HISTORY.md
```
